# BNotes 72 小時 AI 引用與搜尋可見度抽查 SOP

日期：2026-05-20

## MD 圓桌會議結論

建議固定執行。BNotes 的目標是成為華語世界最具公權力、公信力的咖啡知識部落格，因此每篇文章完成升級後，不只要確認內容品質，也要確認它是否逐步被搜尋引擎看見、被 AI 系統理解、被讀者進入與引用。

72 小時抽查不是排名焦慮，而是「可見度雷達」。它的目的不是每三天要求排名大幅上升，而是提早發現索引、搜尋意圖、AI 引用與內部連結方向是否正確。

## 抽查節奏

- 固定每 72 小時執行一次。
- 每次優先抽查最近修正 / 發布 / 升級的文章。
- 每次最多聚焦 5 到 10 篇，避免報表過大而失去行動性。
- 每次產出一份 MD 圓桌會議摘要，標示是否需要升級決策。

## 必查項目

1. **部署與結構健康**
   - P0 / P1 是否維持 0。
   - sitemap / RSS 是否包含正確文章。
   - H1、canonical、Article schema、Breadcrumb、分享資料是否正常。

2. **Google Search Console 信號**
   - 哪些文章已有 impressions。
   - 哪些 query 平均排名接近 8 到 20 名。
   - 哪些文章有曝光但點擊不足。
   - 是否有新索引問題、sitemap 警告或異常下降。

3. **GA4 讀者行為**
   - page views / active users。
   - article_read_complete。
   - scroll / click / newsletter_signup 等事件。
   - 有人看但互動不足的文章，標記為「閱讀節奏或導讀需檢查」。

4. **公開搜尋可見度**
   - 使用 `site:bnotescoffee.com` 檢查主題文章是否被 Google 看見。
   - 使用自然關鍵字抽查是否開始出現 BNotes。
   - 比對是否仍發生「替身文章」現象，也就是 Google 用錯文章回答正確主題。

5. **AI 引用與 GEO 信號**
   - 檢查文章是否有清楚答案句、FAQ、表格、來源與可引用段落。
   - 對核心問題做 AI 引用抽查；若無法直接查詢某 AI 平台，必須標記「本輪不可驗證」，不可臆測。
   - 若 AI / 搜尋抓錯文章，回饋給 B8 / B1 重新檢查標題、導讀卡、FAQ、內部連結與 schema。

## 升級門檻

以下情況需帶回 MD 圓桌會議：

- P0 或 P1 不為 0。
- 重要文章 72 小時後完全沒有被 sitemap / GSC 看見。
- 核心關鍵字仍 0 結果，且站內已有對應文章。
- Google 或 AI 明顯用錯文章回答主題。
- 文章有曝光但互動不足，且可能是導讀卡、標題或閱讀節奏問題。
- 發現可能影響公信力的事實、來源或未來新聞問題。

## 團隊分工

- BNotes PGM：統整報告、判斷是否需要升級。
- B8 SEO/GEO：搜尋、AI 引用、GSC 關鍵字與 query 判斷。
- B9 技術發布：P0/P1、schema、sitemap、RSS、部署健康。
- B1 知識架構：判斷文章是否真的能被引用、是否需要補來源或底層邏輯。
- B3 編輯：根據回饋修正導讀卡、FAQ、段落與延伸閱讀。

## 原則

不追求短期排名幻覺，只追求長期可見度、可引用性與公信力累積。每次 72 小時抽查都必須回到同一個問題：這篇文章是否更容易被讀者找到、被搜尋理解、被 AI 引用，並讓 BNotes 更接近華語咖啡知識權威。
