Module 02 · Cloudflare Pages Deployment
Cloudflare Pages 完整部署指南
BNotes 首選部署平台。掌握以下 4 個關鍵設定，確保靜態部署正確無誤。
CF
Cloudflare Pages 初始設定流程
預計時間 30 分鐘 · 完成後 BNotes Blog 正式上線
30min
SETUP TIME
1
建立 Cloudflare 帳號 + 新增 Pages 專案
前往 dash.cloudflare.com → Pages → Create a project → 選「Direct Upload」（不依賴 GitHub，可手動部署）或「Connect to Git」（自動部署）。Project name 建議：bnotes
必做
2
設定 public/_redirects（SPA 必要）
在專案根目錄的 public/ 資料夾中建立 _redirects 檔案，內容如下。沒有這個設定，重新整理任何非首頁路由都會出現 404。
/* /index.html 200
3
設定 robots.txt + sitemap.xml
在 public/ 建立以下兩個 SEO 必要文件。
User-agent: * Disallow: Sitemap: https://bnotes.tw/sitemap.xml
<?xml version="1.0" encoding="UTF-8"?> <urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"> <url><loc>https://bnotes.tw/</loc><priority>1.0</priority></url> <url><loc>https://bnotes.tw/#/articles/v60-extraction-science</loc><priority>0.9</priority></url> <!-- 每篇文章加一筆 --> </urlset>
4
打包 ZIP 正確格式（關鍵陷阱）
Cloudflare 要求 ZIP 根層直接是 index.html，不能有額外資料夾層次。使用以下 Python 腳本打包：
import os, zipfile dist_dir = "/workspace/bnotes-blog/dist" zip_path = "/workspace/bnotes_deploy.zip" with zipfile.ZipFile(zip_path, 'w', zipfile.ZIP_DEFLATED) as zf: for root, dirs, files in os.walk(dist_dir): for file in files: full_path = os.path.join(root, file) arc_name = os.path.relpath(full_path, dist_dir) # 關鍵：相對於 dist_dir zf.write(full_path, arc_name) print(f"✅ ZIP 打包完成: {zip_path}") print(f" 根目錄文件: {[f for f in os.listdir(dist_dir) if os.path.isfile(os.path.join(dist_dir,f))]}")
⚠️ 陷阱警示：不能用 shutil.make_archive 或系統 zip，否則路徑會變 dist/index.html，Cloudflare 找不到根層 index.html。
5
部署後必做：Purge CDN 快取
每次部署後，前往 Cloudflare Dashboard → 選擇你的網域 → Caching → Configuration → Purge Everything。然後用無痕瀏覽器視窗測試，避免本地快取干擾測試結果。
每次部署必做
6
自訂網域設定（bnotes.tw）
Pages → Custom domains → Add domain → 輸入 bnotes.tw → Cloudflare 自動設定 DNS（若網域在 Cloudflare 管理）或提供 CNAME 記錄（其他 DNS 商）。SSL 憑證自動部署。
自動 HTTPS全球 CDN

📁 BNotes 靜態文章目錄結構（避免 308 重定向）

bnotes-blog/ ├── dist/ # Build 輸出（上傳 Cloudflare） │ ├── index.html # SPA 入口（根層必須直接有） │ ├── assets/ # JS / CSS bundles │ ├── robots.txt │ ├── sitemap.xml │ ├── _redirects # /* /index.html 200 │ └── articles/ # 靜態 HTML 文章 │ ├── v60-extraction-science/ │ │ └── index.html # Cloudflare 對目錄回應 200 ✅ │ ├── taiwan-coe-guide/ │ │ └── index.html │ └── ... └── src/ ├── lib/index.ts # 文章 metadata + 自動發布邏輯 └── pages/ArticleDetail.tsx # STATIC_ARTICLE_MAP + 跳轉邏輯
⚠️ 重要：靜態 HTML 文章必須存為 /articles/slug/index.html 目錄格式，不能用 /articles/slug.html。 Cloudflare 對 .html 結尾路徑會回應 308 重定向，iframe 無法載入，瀏覽器直連也會異常。

import { serve } from "https://deno.land/std/http/server.ts"; import { createClient } from "https://esm.sh/@supabase/supabase-js"; const corsHeaders = { "Access-Control-Allow-Origin": "*", "Access-Control-Allow-Headers": "authorization, x-client-info, apikey, content-type", }; serve(async (req) => { if (req.method === "OPTIONS") return new Response("ok", { headers: corsHeaders }); try { const { email, name, leadMagnet } = await req.json(); const res = await fetch("https://api.resend.com/emails", { method: "POST", headers: { Authorization: `Bearer ${Deno.env.get("RESEND_API_KEY")}`, "Content-Type": "application/json", }, body: JSON.stringify({ from: `${Deno.env.get("FROM_NAME")} <${Deno.env.get("FROM_EMAIL")}>`, to: [email], subject: `☕ 歡迎來到 BNotes 實驗室，${name || "咖啡愛好者"}！`, html: buildWelcomeHtml(name, leadMagnet), }), }); if (!res.ok) throw new Error(await res.text()); return new Response(JSON.stringify({ ok: true }), { headers: { ...corsHeaders, "Content-Type": "application/json" }, }); } catch (e) { return new Response( JSON.stringify({ error: (e as Error).message || String(e) }), { status: 500, headers: { ...corsHeaders, "Content-Type": "application/json" } } ); } });
